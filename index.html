BrightS

A modern x86 kernel, built from the ground up in C. Supports both UEFI (x86_64) and legacy BIOS (i386), with a full TCP/IP stack, multitasking, and multi-language runtime.

View on GitHub Build Guide

Engineered for performance

Every subsystem designed from scratch — memory management, scheduling, networking, and storage — optimized for both modern UEFI and classic BIOS systems.

🖥️

Dual Boot

x86_64 UEFI + i386 BIOS. One kernel, two architectures, one codebase.

🧠

Memory Management

4-level / PAE paging, slab allocator, 8 MB heap, PMEM abstraction.

⚡

O(1) Scheduler

Round-robin with 10-tick preemption, SMP-ready, process groups.

🌐

TCP/IP Stack

VirtIO-Net, DHCP, DNS, HTTP — real protocols, real networking.

💾

Storage Stack

AHCI, NVMe, ramdisk, block device interface, Btrfs filesystem.

🔒

Security

SMEP / SMAP, heap hardening, syscall boundary checking.

🐚

LightShell

History, tab completion, pipes, redirection, job control, env vars.

🧩

Multi-Language Runtime

First-class support for C, C++, Rust, Python, and custom Lang.

🚀

SIMD Optimized

SIMD memcpy / memset, LRU caches (DNS, path, inode, buffer), LTO.

Two architectures, one kernel

BrightS runs on both modern UEFI systems and legacy BIOS hardware, sharing the same core code.

🖥️

x86_64 · UEFI

4-level paging, clang + LLD, ISO image. Built for modern PCs.

🔄

i386 · BIOS

PAE paging, GCC freestanding, floppy image. Built for vintage hardware.
