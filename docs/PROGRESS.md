# BrightS 操作系统 —— 项目进展

## 这是个什么项目？

BrightS 是一个**全新的操作系统**。就像 Windows、macOS、Linux 一样，它可以管理电脑硬件、运行程序、处理文件。只不过它还很年轻，像一个刚学会走路的宝宝。

## 目前支持什么？

### 主体系统（x86_64 架构 — 主流电脑）
已经能在一台正常电脑（64位、UEFI启动）上跑起来了，功能包括：
- **启动**：从 UEFI 引导，进入图形界面
- **显示**：能画文字、简单的图形
- **输入**：键盘可以打字
- **文件系统**：可以读写硬盘上的文件
- **内核功能**：任务调度、内存管理、进程控制等
- **联网**：基本的网络功能（还在开发中）
- **命令行**：BrightS 自己的小 Shell 脚本环境

### 老电脑支持（i386 — 像古董电脑 Samsung NC10）

最近我们加上了对**老式 32 位电脑**的支持，比如 2008 年的上网本。这类电脑：
- 没有 UEFI 固件（只有 Legacy BIOS）
- CPU 不能跑 64 位
- 像 Samsung NC10（Atom N270）就是典型

目前做到的程度：
1. ✅ **BIOS 引导**：能从软盘/USB 启动
2. ✅ **保护模式切换**：从 16 位实模式进入 32 位保护模式
3. ✅ **内核启动**：能跳转到 32 位内核代码执行
4. ✅ **分页基本支持**：内存映射已准备
5. ❌ **完整内核**：还没有把主体系统的所有功能移植过来

## 代码结构 —— 长什么样？

```
BrightS/
├── arch/x86_64/            # 64位架构代码 (主力)
├── sys/                    # 系统内核代码
│   └── kernel/
│       ├── core/           # 内核核心模块
│       ├── fs/             # 文件系统
│       ├── net/            # 网络
│       ├── arch/i386/      # 32位架构代码 (新加的)
│       │   ├── boot/       # 引导程序
│       │   └── entry.asm   # 内核入口
├── drivers/                # 硬件驱动
├── sys/user/               # 用户程序
├── scripts/                # 构建脚本
└── include/kernel/         # 头文件
```

## 下一步做什么？

### 短期（几周内）
- 把完整的 BrightS 内核移植到 32 位
- 让 NC10 能跑起亮机画面
- 简化 i386 引导，去掉调试信息

### 中期（几个月内）
- 32 位下的 AHCI 驱动（硬盘读写）
- 32 位下的 USB 支持
- 网络连接

### 长期
- 降低配置需求，让更多老电脑能跑
- 持续更新优化

## 能在 NC10 上跑了吗？

**能启动，但不能用。** 引导程序工作正常了：
```
BIOS → BrightS i386 → OK → [保护模式] → 内核代码运行
```

但完整的内核功能（图形、文件、网络）还没有移植过来。目前只有最基本的"亮机"功能。

## 图说："电脑开机到系统加载的过程"

```
按电源键
    ↓
BIOS 启动（硬件自检）
    ↓
从 USB/软盘 读取引导程序
    ↓
引导程序加载内核到内存
    ↓
切换 CPU 到 32 位保护模式
    ↓
跳转到内核代码
    ↓
内核初始化（驱动、内存、文件系统）
    ↓
启动 Shell 命令行
```

NC10 的进度已经走到第 5 步，后面几步还在开发中。
