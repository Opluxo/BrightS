# 贡献指南

[EN](CONTRIBUTING.md) · [JA](CONTRIBUTING_ja.md)

## 如何贡献

1. Fork 本仓库
2. `git checkout -b feature/name`
3. 编写代码
4. `make test` — 测试必须全部通过
5. 提交 PR

## 提交格式

```
feat: 新功能
fix:   bug 修复
docs:  文档
refactor: 重构
perf:  性能优化
test:  测试
style: 格式化
ci:    CI/CD
chore: 维护
```

## 代码规范

- 遵循现有代码风格
- 公共 API 使用 `brights_` 前缀
- 函数保持小巧且专注
- 复杂逻辑添加注释

## 许可证

贡献即代表您同意 GPL v2。
