Architecture

Two platforms,
one kernel.

BrightS runs on both x86_64 UEFI and i386 BIOS systems, sharing the same core codebase. No forks, no separate branches — just a single kernel that adapts.

Built with clang + LLD for UEFI targets and i686-linux-gnu-gcc for legacy BIOS, producing a ~176 KB EFI image or a 1.44 MB floppy image.

Performance

Engineered for speed.

An O(1) round‑robin scheduler with 10‑tick preemption keeps processes responsive. The slab allocator manages an 8 MB heap with minimal fragmentation.

SIMD‑optimized memcpy and memset accelerate memory operations. Multi‑layer LRU caches (DNS, path, inode, buffer) front every I/O path.

Link‑time optimization (LTO) is enabled for the UEFI target, producing leaner, faster binaries.

Networking

Real protocols,
real networking.

BrightS ships a complete TCP/IP stack with VirtIO‑Net drivers. DHCP performs a full DISCOVER → OFFER → REQUEST → ACK handshake over UDP, binding to port 68 with timeout and retries.

The DNS resolver builds raw‑byte queries, handles name compression, and parses A‑record responses. The HTTP client resolves DNS, opens TCP connections, and sends GET / POST requests — all from kernel space.

The netget command fetches URLs directly from the shell, no userspace daemon required.

Developer

Multi‑language
by design.

BrightS includes first‑class runtime support for C, C++, Rust, Python, and Lang. Write kernel modules in Rust, script shell sessions in Python, or compile C++ programs directly with the built‑in g++ command.

The LightShell environment features history, tab completion, pipes, redirection, job control, environment variables, and a growing set of built‑in commands including netget, ping, and export.

File systems supported: RAMFS, DevFS, Btrfs, all through a unified VFS2 interface.
